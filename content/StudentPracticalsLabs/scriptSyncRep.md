## Скрипт синхронизации репозиториев

Автоматизация рутинных задач с помощью скриптов - важная часть профессиональной деятельности любого разработчика!

> При создании скриптов следует учитывать особенность, что две одинаковые буквы английского алфавита, написанные в разных регистрах (верхний или нижний) - для **Windows** будут неразличимы, а для **Unux** (Linux и MacOS) - разными символами! Например: 'a' и 'A' - для винды это одни и теже буквы, а для **Linux/MacOS** - разные. Папки `Content` и `сontent` для винды и линукса обрабатываются по-разному.

> Будьте внимательны к генерируемому нейронкой коду, проверяйте пути. Есть риск потерять свои данные из-за некорретный путей при копировании из одной папки в другую!

- При помощи нейросети создать скрипт на **bash** и **powershell**, который синхронизирует выбранный локальный репозиторий с его удалённой копией (команда `git pull`) в корневом каталоге пользователя, <u>скопирует</u> его содержимое с проверкой пути, кроме папки `.git`, в локальную папку другого репозитория с проверкой пути и с заменой имеющихся там файлов и папок с теми же именами.
- Опционально в вашем скрипте можно добавить пропуск или обработку приватных репозиториев и т.п.;
- Опционально можно сделать так, чтобы ваш скрипт выполнялся явно, в окне автоматически запущенного Терминала, для визуального контроля его выполнения;
- Протестировать скрипт внутри Docker-контейнера с **Ubuntu** или **WSL**, виртуальной **Windows** (в **Virtual Box** или **Hyper V**). Для тестирования в указанных ОС придётся в них склонировать два репозитория - мой и ваш, отредактировать пути и запустить ваш скрипт синхронизации. Если синхронизация прошла успешно, то можно поместить его в автозагрузку вашей хостовой ОС для удобства. Перед тестирование в виртуальных ОС не забудьте установить в них **git**!
- Результат (код скрипта с описанием его работы) прислать в виде ссылки на вайл `README.md` с описанием проекта.
- Используйте **нейросети** для помощи с кодом.
