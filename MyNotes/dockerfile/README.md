## Что такое Docker?
**Docker** — это инструмент, позволяющий упаковать приложение и все его зависимости (библиотеки, настройки, среду) в единый изолированный **контейнер**.

**Почему это круто:**
1. **Изоляция:** Разные проекты на одном ПК не конфликтуют.
2. **Переносимость:** "У меня работает" — теперь это работает везде (на сервере, у коллеги, в облаке).
3. **Скорость:** Контейнеры запускаются мгновенно.

---

## Основные понятия
* **Image (Образ):** Неизменяемый шаблон (как ISO-образ или чертеж).
* **Container (Контейнер):** Запущенный экземпляр образа (работающее приложение).
* **Docker Hub:** Магазин/реестр готовых образов (как GitHub, но для контейнеров).
* **Volume (Том):** Внешнее хранилище для данных, которые должны выжить после удаления контейнера.

## Шпаргалка по командам CLI

### Контейнеры
```bash
# Запустить контейнер (скачает образ автоматически)
docker run -d -p 8080:80 --name my-web nginx
```
```bash
# Список запущенных контейнеров
docker ps
```
```bash
# Список всех контейнеров (включая остановленные)
docker ps -a
```
```bash
# Остановить / Запустить контейнер
docker stop <ID_или_имя>
docker start <ID_или_имя>
```
```bash
# Удалить контейнер
docker rm -f <ID_или_имя>
```
```bash
# Зайти в терминал запущенного контейнера
docker exec -it <ID_или_имя> sh
```

## Образы
```bash
# Список локальных образов
docker images
```
```bash
# Удалить образ

```
```bash
#Полная очистка мусора (неиспользуемые образы и контейнеры)
```

## Создание своего образа (Dockerfile)
**Dockerfile** — это сценарий сборки вашего приложения. Создайте файл Dockerfile в корне проекта:
```bash
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

## Сборка и запуск:
1. Собрать: docker build -t my-app .

2. Запустить: docker run -p 3000:3000 my-app

## Работа с данными (Volumes)
Контейнеры "забывчивы". Чтобы сохранить данные БД или загруженные файлы, используйте тома:
```bash
# Монтируем папку хоста в папку контейнера
docker run -v /my/data:/var/lib/mysql mysql
```

## Оркестрация (Docker Compose)
Если у вас связка "Бэкенд + БД", используйте файл docker-compose.yml:
```bash
version: '3.8'
services:
app:
build: .
ports:
- "3000:3000"
depends_on:
- db
db:
image: postgres:15-alpine
environment:
POSTGRES_PASSWORD: mysecretpassword
volumes:
- pgdata:/var/lib/postgresql/data
volumes:
pgdata:
```

## Управление:
```bash
docker-compose up -d — запустить всё.
```
```bash
docker-compose down — остановить и удалить всё.
```
```bash
docker-compose logs -f — просмотр логов.
```
## Лучшие практики
1. Используйте .dockerignore: Всегда исключайте node_modules, .git и папки сборки.

2. Alpine-образы: Ищите образы с тегом -alpine (они весят намного меньше).

3. Версионность: Вместо latest пишите конкретную версию (напр. python:3.10-slim).

4. Слои: Сначала делайте COPY package.json и RUN npm install, а только потом COPY . . — это ускоряет сборку.
